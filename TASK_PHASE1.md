# 任务单:黄金段落库校准循环(Phase 1)

## 背景

Phase 0 结论:统计层可产出全局基准线和 n-gram 自检,但场景分类区分度不足。因此体系调整为:**few-shot 黄金段落库为核心,统计层只做写完后的量化验收**。

本阶段目标:通过"CC 原创试写 → 自动验收 → 人工评分 → 修正规则"的循环,沉淀一批经人工认可的**原创示范段落**(黄金段落库),作为以后正式写作的 few-shot 弹药。

## 安全边界(不变)

- 所有试写内容必须是**全新原创**的人物和情节,严禁改写、续写、扩写语料中的任何情节和人物。
- 每篇试写完成后必须通过 n-gram 自检。**底库 = longzu.txt + 龙与少年游_utf8.txt 合并**(龙族1.txt 内容与合订本重复,不需加入):
  ```bash
  cat corpus/longzu.txt corpus/龙与少年游_utf8.txt > corpus/ngram_base.txt
  ```
- golden/ 目录中的一切内容都是原创产物,文件头标注 `<!-- 原创示范,非原著引用 -->`。

## 语料使用备注

- 统计基准以 **baseline_longzu.json** 为唯一标准。
- 龙与少年游(散文集)**不得混入基准**(句长 43.2 字、零感叹号,是完全不同的体裁),仅作为查重底库的一部分。

## 执行规则

- Step 0-2 自动执行;Step 3 需要我人工评分,产出后停下等我。
- 错误自主修复最多 3 次,失败则停止汇报。
- 进度追加到 progress.md。

## Step 0:负面清单(anti-patterns)

创建 `rules/anti_patterns.md`,内容分两部分:

**A. 通用 AI 腔病灶(直接写入,不依赖语料):**
- 短句无脑堆砌(连续 4 个以上 ≤8 字的句子,且无节奏意图)
- "仿佛/宛如/像是/如同"高频堆叠(基准线见 B 部分)
- 每段结尾强行升华/抒情
- 排比滥用(一段内超过 1 组三连排比)
- 形容词轰炸(一个名词前挂 2 个以上形容词)
- 空洞哲理句("这就是青春""命运就是如此"类的直白总结)
- 对话标签单调(全部是"XX说",或反向全部花式动词"低吼道/轻叹道")
- **"欲言又止"句式（"想说什么却没说/翻出来看了一眼又没说"类）全篇最多出现 1 次，且必须搭配具体信息点（读者能推断出他到底想说什么），否则视为无信息量的空转，直接删除**
- **单段感叹号数量上限：每段 ≤1 个，全篇平均值达标不能掩盖局部堆积**
- **禁止用"你知道的/不重要/你清楚的"这类空话搪塞关键设定信息。核心冲突（谁、要什么、为什么）必须有至少一处具体锚点，不能全篇靠语气代替内容**

**B. 量化红线(基于 Phase 0 实测基准,直接使用以下数字):**
- 平均句长允许区间:**20-30 字**(基准 24.9)
- 句长分布:四档(≤8 / 9-20 / 21-40 / >40 字)各自允许区间 **17%-35%**(基准约 27/22/27/24);**短句占比超过 35% 即判定碎片化,一票否决**
- 平均段落长下限:**40 字**(基准 58.4;低于 40 说明段落被切碎)
- "仿佛/宛如/像是/似乎/如同"合计频率上限:**每万字 20 次**(基准合计约 9-13,从 baseline_longzu.json 读取精确值;试写篇幅短,按比例折算)
- 感叹号频率允许区间:**每万字 25-62**(基准 41.29;高压场景取上半区,日常场景取下半区)
- 省略号频率上限:**每万字 35**(基准 22.32)
- 对话字数占比:高压场景 **15%-30%**(基准战斗章 23.2%),日常场景 **30%-50%**(基准日常章 40.9%)

**C. 分句级红线(防逗号粘连作弊,第2轮反馈新增):**
使用前必须先用更新版 style_stats.py 重跑 corpus/longzu.txt,取得三项分句指标的江南基准值,再设定:
- 平均分句长(逗号切):不低于基准值的 80%
- 极短分句占比(≤5字):不高于基准值的 1.3 倍
- **粘连句占比(≥4分句且均长≤7字):不高于基准值的 1.3 倍,超标一票否决**——句号级指标全绿但分句级超标,同样判定为碎片化

注:试写样本仅 1500-3000 字,频率类指标按每万字折算后对照;单篇样本量小,允许 ±30% 的额外浮动,但短句占比红线(35%)和粘连句红线不放宽。

## Step 1:试写规范 v0

创建 `rules/writing_spec.md`,包含:
1. 叙事视角要求:自嘲式第一人称,行动与语言的反差感
2. 节奏要求:轻盈(日常/调侃)与沉重(命运/真相)交替
3. 长段落搭建方式:动作链条或环境+心理交替,禁止句号硬切碎拼
4. 写作流程强制要求:**先出分镜稿,再按分镜稿落笔**
   - 分镜稿格式:镜头号/景别/视点/内容概要/时长感(拉长细写 or 快切)/预计行数
5. 引用 anti_patterns.md 作为禁止项

## Step 2:试写循环(每轮自动执行)

每轮产出一篇 **1500-3000 字的完整场景**(不是片段——碎片化问题只在长文本暴露)。

试写题目池(全新原创,依次使用):
1. 【高压】深夜地下停车场,主角被迫与曾是好友的对手正面冲突
2. 【日常】大学食堂,主角和损友讨论一件小事,结尾埋一个不安的伏笔
3. 【情感】天台告别戏,想说的话最终没说出口
4. 【高压】雨夜追逐,主角在逃跑途中做出一个改变立场的决定
5. 【日常→沉重】平静的家庭晚餐,一句无心的话揭开旧伤疤

每轮流程:
1. 按 writing_spec.md 出分镜稿,存 `drafts/round<N>_shots.md`
2. 按分镜稿写正文,存 `drafts/round<N>_draft.txt`
3. 自动验收:
   ```bash
   python3 scripts/style_stats.py drafts/round<N>_draft.txt drafts/round<N>_stats.json
   python3 scripts/ngram_check.py corpus/ngram_base.txt drafts/round<N>_draft.txt 10
   ```
4. 对照 anti_patterns.md 的量化红线逐项检查:
   - 任何一项超出红线 → 定位问题段落,重写,重跑验收(最多 3 次)
   - n-gram 未通过 → 重写命中片段,重跑
5. 全部通过后,生成 `drafts/round<N>_report.md`:统计对照表(本篇 vs 基准线)+ n-gram 结论 + 分镜稿执行情况自评
6. **停下,提交给我人工评分**

## Step 3:人工评分(我来做)

我会按以下维度打分(1-5)并标注具体位置:

| 维度 | 说明 |
|---|---|
| **剧情清晰度** | **一票否决项。读完能否复述"发生了什么/为什么/谁想要什么"；靠氛围掩盖设定空缺一律扣到1分** |
| 句子节奏 | 长短交替是否自然,有没有碎 |
| 对话腔调 | 有没有梗感/互怼感,是否像真人说话 |
| 金句质感 | 抒情句是否克制、有留白（留白=读者能脑补出确切答案而作者选择不说），还是含糊（=作者自己没想清楚，用氛围搪塞） |
| 画面感 | 分镜是否体现在文字里,镜头切换是否流畅 |
| 整体气质 | 盲测像不像目标文风 |

**判定规则更新：剧情清晰度 ≤2 分，无论其他维度多高，本轮判定为不通过。**

## Step 4:规则修正 + 黄金库沉淀(收到我的评分后执行)

1. 针对每条低分标注,修改 writing_spec.md / anti_patterns.md 的对应条款(修改处注明"第N轮反馈")
2. 总分 ≥20 且无单项 ≤2 的段落 → 整篇存入 `golden/`,按题目类型命名(如 `golden/highpressure_01.txt`)
3. 总分不足但有局部亮点 → 摘出亮点段落(仅摘 CC 自己写的内容)存入 `golden/fragments/`
4. 进入下一轮试写,新一轮必须在 prompt 中携带:writing_spec.md + anti_patterns.md + golden/ 中已有的全部内容作为 few-shot

## 终止条件

连续两轮总分 ≥20 → Phase 1 完成,汇报并等待 Phase 2 指令(封装 SKILL.md + 正式创作流程)。
5 轮后仍未达标 → 停止,输出各轮评分趋势和你认为的瓶颈,等我决策。

## 最终交付(每轮)

1. round<N>_draft.txt 正文
2. round<N>_report.md(统计对照 + 自检结论)
3. 一句话:本轮相对上一轮改了什么
